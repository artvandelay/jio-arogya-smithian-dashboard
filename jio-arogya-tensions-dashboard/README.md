# Jio Arogya Strategic Tensions Explorer

This dashboard is a decision artifact for senior leadership.

It is not a claim register, FAQ table, or operating checklist. It models four load-bearing strategic levers and shows the consequences of degrading any one of them across the five Jio Arogya operating layers.

## Purpose

The core question is:

> Does Jio Arogya remain a compounding health operating system if AI, access, data depth, or governance are funded down?

The dashboard is designed to surface the structural consequence without making "AI underfunding" the headline. The user flips levers and sees the cascade.

## Levers

1. **AI Autonomy + Investment**
   - Full: clinical-grade Personal Agent + Clinical AI as operating spine
   - Degraded: basic triage / underfunded AI

2. **Delivery Capex + Access**
   - Full: CHW + home + thin physical layer
   - Degraded: asset-heavy clinic/pod rollout

3. **Data Acquisition Depth**
   - Full: deep baseline + longitudinal biology
   - Degraded: claims/labs only to save cost

4. **Governance + Trust Architecture**
   - Full: independent Trust + hard privacy walls
   - Degraded: commingled operating data

## Operating Layers

- Insurance Base
- Proactive Engagement Stack
- Navigation Layer
- Data Architecture + Biobank
- Medical + Scientific Asset

Each layer can be green, amber, or red:

- **Green:** the layer compounds
- **Amber:** the layer still works but loses strategic advantage
- **Red:** the mechanism that makes the model economic or defensible breaks
